逛某个网站时发现 F12 打不开，通过浏览器菜单进去开发者工具也会在几秒后跳转到主页，甚至在打开网页之前先打开开发者工具也会触发跳转。挺好奇怎么做到的，搜了一下发现是这个库：https://github.com/theajack/disable-devtool

![image-20260412153137807](https://picgo.19991029.xyz/image-20260412153137807.png)

## 实现原理

这个库用了多种探测器来检测开发者工具是否打开：

- **RegToString**：正则检查
- **DefineId**：基于 DOM ID 检测
- **Size**：基于窗口大小检测
- **DateToString**：Date.toString 检查
- **FuncToString**：Function.toString 检查
- **Debugger**：断点检测（仅 iOS Chrome 真机有效）
- **Performance**：基于日志的性能检测
- **DebugLib**：检测第三方调试工具（eruda、vconsole）

核心思路是定时循环检测（默认 200ms），多维度验证开发者工具是否打开，打开就执行回调跳转。

## 为什么先打开开发者工具也会被检测到

因为库会持续监控各种指标，比如窗口大小变化、DOM 属性被修改等。即使开发者工具是先打开的，页面加载后检测到这些异常状态也会触发跳转。

## 开发者绕过方式

支持通过 URL 参数绕过：

1. 设置密钥（如 `abc123`）
2. 用库提供的 MD5 方法加密
3. 访问时加参数 `?ddtk=abc123` 即可绕过

## 使用方法

NPM 安装：

```bash
npm i disable-devtool
```

```javascript
import DisableDevtool from 'disable-devtool';

DisableDevtool({
    url: 'redirect-url',  // 跳转页面
    disableMenu: true,    // 禁用右键菜单
    ondevtoolopen: (type) => {
        console.log('开发者工具被打开，类型：', type);
    }
});
```

Script 标签引用：

```html
<script 
    disable-devtool-auto 
    src='https://cdn.jsdelivr.net/npm/disable-devtool'
></script>
```

## 总结

这个库挺有意思的，虽然不能完全防止技术用户绕过，但对普通用户来说足够了。做前端保护可以考虑用这个。
